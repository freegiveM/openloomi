# Unified Inbox Auto-Fetch + Context-Aware Draft Generation

## Loomi Sits on Your Desktop, Watches the Unified Inbox, Hands You Replies

Professionals lose 2–3 hours a day to email and IM — most of it spent re-reading threads to remember what's been decided before typing a reply. Most AI inbox tools either auto-send blindly (low accuracy) or ask you to paste context every time (high friction). Loomi — your **attention agent** — does neither. It lives on your desktop, fetches your **unified inbox** — Gmail, Slack, Telegram, Lark, iMessage, the lot — on a quiet schedule, builds a compact context map of each thread, and drafts only the replies that actually need one. When a draft is ready, a small bubble appears at the corner of your screen — tap to review, edit if you like, send when you're ready.

## How Loomi Handles a New Thread

Loomi treats every incoming message as a decision that may need your attention. It only surfaces the ones that match what you've taught it to watch for.

| Step            | What Loomi does                                                                                                             |
| --------------- | --------------------------------------------------------------------------------------------------------------------------- |
| 👀 Watches      | Every 30 min, Loomi fetches new mail from your Gmail + new DMs from Slack / Telegram / Lark / iMessage, all on your desktop |
| 🧠 Maps context | Loomi reads the whole thread — topics, past decisions, pending items, sender's actual question                              |
| ✍️ Drafts       | Generates a reply in your voice, grounded in the context map (not the raw thread)                                           |
| 💬 Nudges       | Pops a small bubble: _"Reply ready for Sarah — Q3 OKR status"_                                                              |
| ✅ You decide   | Tap to review, edit if needed, send with one click — Loomi never auto-sends                                                 |

## Loomi Remembers — openloomi-memory Powers Every Draft

Every draft Loomi produces is grounded in your **global memory**, not just the thread in front of it. Loomi runs the `openloomi-memory` skill on every pass — it pulls context **before** drafting, then writes back what it learned **after** you send. Two directions, one memory layer.

| Direction          | What Loomi does                                                                                                                                                                                                             |
| ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 🔍 **Read first**  | Before drafting, Loomi queries your `~/.openloomi/data/memory/` for this sender's profile, the project this thread belongs to, and any past decision you've made with them. Only the relevant slice — never the whole graph |
| 💾 **Write after** | After you tap Send, Loomi records the sender, the project association, the decision you just made, and any new entity surfaced in the reply — so the next email lands with sharper context                                  |

What lives in memory under `~/.openloomi/data/memory/`:

| Subdirectory | What Loomi stores here                                                                |
| ------------ | ------------------------------------------------------------------------------------- |
| `people/`    | One profile per frequent contact — name, role, project ties, last interaction summary |
| `projects/`  | One note per active project — owner, status, the open thread cluster around it        |
| `insights/`  | Auto-extracted entities, decisions, and events distilled from past chats and emails   |

You don't have to curate any of this. A daily analytics refresh surfaces the contacts and projects you actually engage with; a weekly compaction archives stale entries instead of deleting them, so context never silently disappears. Loomi's draft for Sarah Chen — whether it's an email reply or an IM reply — knows who Sarah is, what Q3 OKR thread belongs to, and which open items you've discussed with her across every channel. Without you ever typing a note.

> 💡 The same memory layer is what `IM_REPLY`, `review_pr`, and the rest of the Loop card family read from. When Sarah emails you about Q3 OKR, the IM_REPLY draft to her on Slack already knows it's the same Sarah. One memory, every channel.

## Setting Up Loomi for Your Unified Inbox

### Connect Your Accounts

One connector call. Works in OpenLoomi Desktop, the Claude plugin, or the Codex plugin — invoke the `openloomi-connectors` skill in chat:

```
use openloomi-connectors to connect Gmail
```

Once connected, you can also **chat with Loomi about any message** — ask about a specific thread, summarize a long back-and-forth, or pull details from any message in your unified inbox. Loomi remembers it all.

## The Loop Decision Card: DRAFT_REPLY

When Loomi notices a thread that genuinely needs your reply, it surfaces a typed decision card — not a raw blob of generated text. For email, the card type is **`DRAFT_REPLY`**: signal (gmail), type (email), received time, and Loomi's confidence score, plus the full reasoning trace from the **From Loomi** panel — _why_ Loomi thinks this needs your attention.

The setup is identical to email — connect the messaging connector, tell Loomi what to watch for, and approve each send. Loomi never auto-messages; every reply waits for your tap.

## Why Loomi Beats Brute-Force AI Inbox Tools

Most AI inbox tools dump the entire thread into the model — expensive, slow, and often counterproductive. Loomi works differently: it builds a **compact context map** first (what was decided, what's still open, what the sender actually needs) and drafts from that map, not from raw text.

| Aspect            | Before                                | With Loomi                                        |
| ----------------- | ------------------------------------- | ------------------------------------------------- |
| Reply review time | 10–15 min/thread to recall context    | Tap a bubble, read a card, send                   |
| Draft accuracy    | Generic auto-replies miss context     | Thread-aware drafts that read like you wrote them |
| Token usage       | Full thread pasted every time         | Compact context summary, ~40–60% fewer tokens     |
| Setup friction    | Manual copy-paste every time          | One-time connect, then Loomi runs                 |
| Cross-channel     | Switch tools to reply on each channel | One unified queue — same card flow, every channel |
| Control           | Full auto-send (risky) or manual only | Human-in-the-loop approval, every single time     |

## How to Set Up

See [Setting Up Loomi for Your Unified Inbox](#setting-up-loomi-for-your-unified-inbox) above — one `openloomi-connectors` call per account and Loop takes over. The default cadence is every 30 minutes; tune it in **Settings → Loop → Scan frequency**.
